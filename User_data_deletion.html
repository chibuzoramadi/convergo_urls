Convergo User Data Deletion Policy

Last Updated: December 9, 2025

Meta Data Deletion Callback Compliance

This policy satisfies Meta's Platform Terms requirement for user-initiated data deletion. Users can permanently delete their Convergo account and all associated data through our dashboard or by contacting support. Meta apps receive deletion callbacks via webhook. [web:42][web:52]

How to Delete Your Data

Self-Service Account Deletion (Recommended)

  1. Log in to your Convergo dashboard
  2. Navigate to Account Settings → Delete Account
  3. Review what will be deleted (see below)
  4. Click "Permanently Delete My Account"
  5. Confirm via email verification link (expires in 24 hours)
  6. Receive deletion confirmation email with callback details

Track Deletion Status

Check deletion progress: convergo.com/account/deletion-status
Enter your confirmation code from the deletion email.

What Gets Deleted

Immediate Deletion (within 1 hour):
  • All account data (email, name, business details, hashed password)
  • WhatsApp conversations and message history
  • OpenAI prompts, replies, and configuration settings
  • Dashboard analytics and usage data
  • Meta OAuth tokens and WhatsApp Business Account connections
Backup Purge (within 30 days):
  • Database backups containing user data
  • AWS S3 snapshots and RDS automated backups

What May Be Retained (Pseudonymized)

For legal compliance, security, and analytics:

  • Anonymized usage aggregates (no user identification possible)
  • Security event logs (IP addresses masked after 30 days)
  • Fraud prevention records (minimal data, 90-day maximum)

Third-Party Data Handling

  • Meta: Receives deletion callback via webhook, revokes app permissions
  • OpenAI: Deletes processing logs per their Data Processing Addendum
  • AWS: Account data removed from EC2/RDS/S3 storage
  • Payment Processors: Contact provider directly for billing data deletion

Manual Deletion Requests
